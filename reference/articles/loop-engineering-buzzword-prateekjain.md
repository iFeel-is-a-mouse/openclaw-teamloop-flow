## Opening

If you have been following AI-assisted development for the last two years, you have probably noticed one pattern: every few months, the industry finds a new word for the same larger shift. First, everyone was talking about prompt engineering. Then the conversation moved to context engineering. After that, harness engineering started appearing in discussions around AI agents, DevOps, and controlled execution environments. Now the latest term being pushed around is **loop engineering**.

This discussion did not appear randomly. One of the posts that sparked it came from Peter Steinberger, where he said developers should not be prompting coding agents anymore, but should be designing loops that prompt agents.

That line became popular because it captured something many developers are already feeling while using tools like Claude Code, Cursor, Codex, Windsurf, GitHub Copilot agent mode, and other coding agents.
